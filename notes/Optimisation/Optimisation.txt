# GameMaker has two ways of compiling:

## Virtual Machine (VM)

- This will run your code in an intermediate form ("Bytecode") which the loader, linker, and the
build (runner) will then execute when you launch your game.

- Slow... in comparison to YYC. (But still very fast)

## Yoyo Compiler (YYC)

- This transpiles GameMaker Language code into C++, then as the name suggests: compiles / builds
that C++ programme.

- Faster than VM, but slower than self-written native code.

** NOTE(s): **

> Usually, you'd wanna stick to VM unless you got some heavy stuff.
Please don't mess with C++ if you're unsure.

> And another reason is maybe it'll bloat your storage (Visual Studio + C++). Which is redundant for a game that's just a 
square running around the screen.
